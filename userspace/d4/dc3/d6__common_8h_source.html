107 /* RFC 4704 "The DHCPv6 Client FQDN Option"
108  * uint16 option-code OPTION_CLIENT_FQDN (39)
109  * uint16 option-len 1 + length of domain name
110  * uint8 flags
111  * char[] domain-name partial or fully qualified domain name
112  *
113  * Flags format is |MBZ|N|O|S|
114  * The "S" bit indicates whether the server SHOULD or SHOULD NOT perform
115  * the AAAA RR (FQDN-to-address) DNS updates. A client sets the bit to
116  * 0 to indicate that the server SHOULD NOT perform the updates and 1 to
117  * indicate that the server SHOULD perform the updates. The state of
118  * the bit in the reply from the server indicates the action to be taken
119  * by the server; if it is 1, the server has taken responsibility for
120  * AAAA RR updates for the FQDN.
121  * The "O" bit indicates whether the server has overridden the client's
122  * preference for the "S" bit. A client MUST set this bit to 0. A
123  * server MUST set this bit to 1 if the "S" bit in its reply to the
124  * client does not match the "S" bit received from the client.
125  * The "N" bit indicates whether the server SHOULD NOT perform any DNS
126  * updates. A client sets this bit to 0 to request that the server
127  * SHOULD perform updates (the PTR RR and possibly the AAAA RR based on
128  * the "S" bit) or to 1 to request that the server SHOULD NOT perform
129  * any DNS updates. A server sets the "N" bit to indicate whether the
130  * server SHALL (0) or SHALL NOT (1) perform DNS updates. If the "N"
131  * bit is 1, the "S" bit MUST be 0.
132  *
133  * If a client knows only part of its name, it MAY send a name that is not
134  * fully qualified, indicating that it knows part of the name but does not
135  * necessarily know the zone in which the name is to be embedded.
136  * To send a fully qualified domain name, the Domain Name field is set
137  * to the DNS-encoded domain name including the terminating zero-length
138  * label. To send a partial name, the Domain Name field is set to the
139  * DNS-encoded domain name without the terminating zero-length label.
140  * A client MAY also leave the Domain Name field empty if it desires the
141  * server to provide a name.
142  */
143 #define D6_OPT_CLIENT_FQDN 39
