8 /*
9  * Explanation of cancellation bits:
10  *
11  * tcbCancelEnableBit and tcbCancelAsyncBit should be self-explanatory,
12  * they are set if cancellation is enabled, or asynchronous, respectively.
13  *
14  * tcbCancelTriggerBit is set whenever a cancellation is triggered, which is
15  * in pthread_cancel() or in the signal handler. This bit is used by
16  * pthread_testcancel() to check whether a cancellation has been requested,
17  * and also by cancellable syscalls.
18  *
19  * tcbCancelingBit is set when a cancellation is currently being handled. This
20  * is to avoid a situation in which a cancellation handler gets interrupted by
21  * a SIGCANCEL and a second cancellation handler gets executed on top of the
22  * previous one. Right now this cannot happen, since we stay in signal handler
23  * context when canceling/exiting. In the future this might be done outside
24  * of a signal handler, in which case we shouldn't restart the cancellation process.
25  *
26  * tcbExitingBit is set when the thread starts the exit procedure. Currently
27  * this is just an exit, but in the future this will be a stack unwinding
28  * procedure, which shouldn't be reentered. Not currently set anywhere,
29  * may be done so in the future.
30  *
31  * TODO(geert): update this comment when we do unwinding in the exit procedure.
32  */
