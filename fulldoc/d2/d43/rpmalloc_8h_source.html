144  // aligned to the rpmalloc span size, which will always be a power of two.
145  // Optionally the function can store an alignment offset in the offset variable
146  // in case it performs alignment and the returned pointer is offset from the
147  // actual start of the memory region due to this alignment. The alignment offset
148  // will be passed to the memory unmap function. The alignment offset MUST NOT be
149  // larger than 65535 (storable in an uint16_t), if it is you must use natural
150  // alignment to shift it into 16 bits. If you set a memory_map function, you
151  // must also set a memory_unmap function or else the default implementation will
152  // be used for both. This function must be thread safe, it can be called by
153  // multiple threads simultaneously.
154  void *(*memory_map)(size_t size, size_t *offset);
156  // If release is set to non-zero, the unmap is for an entire span range as returned by
157  // a previous call to memory_map and that the entire range should be released. The
158  // release argument holds the size of the entire span range. If release is set to 0,
159  // the unmap is a partial decommit of a subset of the mapped memory range.
160  // If you set a memory_unmap function, you must also set a memory_map function or
161  // else the default implementation will be used for both. This function must be thread
162  // safe, it can be called by multiple threads simultaneously.
